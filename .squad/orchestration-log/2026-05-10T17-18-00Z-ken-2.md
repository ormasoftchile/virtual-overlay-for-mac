## Brief

Diagnosed and fixed multi-display CGS bug in v1.2 implementation.

## The Bug

Cristian's multi-display setup showed all overlays with the same Space name (cross-display collision). Hypothesis confirmed: `CGSGetActiveSpace` is global to the keyboard-focused display, not per-display. On multi-monitor setups, every overlay resolved to the focused display's Space ID, causing stored names to cross-wire between physical displays.

## The Fix

Replaced `CGSGetActiveSpace` with per-display `CGSManagedDisplayGetCurrentSpace(connection, displayUUID)` for each overlay. Fallback chain:
1. Per-display `CGSManagedDisplayGetCurrentSpace` → 
2. Global `CGSGetActiveSpace` (compat fallback) → 
3. Public heuristic fingerprint

Each tier emits stderr diagnostics.
